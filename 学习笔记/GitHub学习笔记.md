Git&GitHub 零基础必学 10 条核心命令实战学习笔记
📌 笔记前言
全套适配新手入门，贴合本地Git+远程GitHub全流程实操，覆盖日常单人开发、基础云端同步全场景，无冗余复杂参数，全程落地可用，适配初学练手、职场基础实操刚需。
通用前置守则：所有操作全程在Git Bash终端执行；修改文件前后、每执行2步Git操作，必查仓库状态，规避90%新手报错问题。

---
✅ 命令 1：git status —— 仓库实时状态监控面板（新手万能前置命令）
🔹 核心作用
一键查看本地完整仓库实时状态：精准识别已修改文件、暂存区待提交文件、未被Git追踪的新建空白文件，全程可视化管控仓库数据，是所有Git操作的前置必用命令。
🔹 标准语法（零参数，直接套用）
git status
🔹 完整实战分步演示
1. 本地仓库内新建文件：命名为学习笔记.md，随意录入内容后保存，不执行任何其他操作；
2. 终端执行命令，页面显示红色文件名，标注Untracked files，释义：全新文件，Git未收录版本管控；
3. 后续执行add命令归集文件后，再次执行本命令，文件名变为绿色，标注Changes to be committed，释义：已存入暂存区，等待归档提交；
4. 全套提交流程走完后，再次执行本命令，提示nothing to commit, working tree clean，释义：所有修改已归档，工作区无冗余未处理内容，仓库干净合规。
🔹 新手专属避坑提醒
- 高频刚需命令，修改代码、执行推送/拉取、回滚版本前后，必须优先执行；
- 牢记极简配色规则：红色=未进暂存区，绿色=已入暂存区，一秒读懂终端状态。

✅ 命令 2：git add —— 工作区修改归集至暂存区（提交必经前置步骤）
🔹 核心作用
将本地工作区中新建、编辑、修改的文件，批量/精准归集到Git专属暂存区，是后续commit归档提交的唯一前置通道，无此步骤无法生成正式版本记录。
🔹 三套标准常用语法（按需选用，新手优先精准模式）
1）精准添加单个文件（新手首选，安全不踩坑）
git add 文件名.后缀
示例：git add 学习笔记.md、git add README.md
2）一键归集全部修改文件（熟练后高效复用）
git add .
⚠️ 关键备注：末尾为英文句号，前方必须带空格，覆盖当前文件夹所有新增、改动文件。
3）批量归集同类型后缀文件（批量办公刚需）
git add *.md
释义：一键归集仓库内所有后缀为.md的markdown格式文件。
🔹 新手专属避坑提醒
- 禁止盲目套用git add .：仅需提交部分文件时，优先用单文件精准添加，避免误归集本地测试文件、私密账号密码配置文件；
- 全新空白/新建文件，必须先执行add命令，Git才会开启版本追踪，否则永久无修改归档记录。

---
✅ 命令 3：git commit —— 暂存区归档生成永久本地版本（版本核心核心）
🔹 核心作用
将暂存区内所有归集完成的修改内容，正式归档锁定到本地Git仓库，自动生成唯一不可篡改哈希版本号，永久留存修改日志，后续可随时回溯、调取、还原对应代码版本，是Git版本管控核心内核命令。
🔹 必用标准基础语法（职场强制规范格式）
git commit -m "本次修改清晰备注说明文字"
⚠️ 强制参数：-m 不可省略；备注文字必须用英文半角双引号包裹，中文引号直接报错中断执行。
🔹 两套进阶省力语法
1）跳过add，直接提交已有跟踪文件（仅适配旧文件修改，新建文件无效）
git commit -am "直接提交：优化页面基础样式布局"
2）修正上一条写错的提交备注（无需重新提交代码）
git commit --amend
🔹 完整实战落地流程
已通过add命令归集学习笔记.md → 终端执行提交命令 → 自动生成哈希版本号，同步展示修改文件数、新增/删除代码行数 → 搭配git status核验，提示工作区干净即提交成功。
🔹 新手+职场双重避坑提醒
- 提交备注严禁敷衍：禁止填写111、修改、已更新等无意义文字，规范标注功能场景，方便后期团队协作回溯排查问题；
- 单次commit绑定单一独立功能：新增笔记、修复bug、优化样式分开提交，不要批量混杂无关修改，降低版本管控难度。

✅ 命令 4：git log —— 调取本地提交历史台账（版本回溯必备工具）
🔹 核心作用
一键调取本地仓库全量归档提交记录，完整展示每一个版本的哈希唯一版本号、提交操作人员、精准提交时间、功能备注说明，回溯还原旧代码、排查版本故障时，必用此命令调取溯源凭证。
🔹 三套新手友好极简语法
1）完整查看全量提交历史（默认原生模式）
git log
2）新手首选：极简一行展示（清爽不杂乱）
git log --oneline
效果：仅展示版本号前7位核心编码+提交备注，无冗余杂项信息。
3）定向查看最近指定次数提交记录
git log -3
释义：数字3可自由替换，按需查看最近2条、5条提交日志。
🔹 实操结果示例展示
a1b2c3d (HEAD -> main) 新增Git十条核心命令全套学习笔记
e4f5g6h (origin/main) 优化README补充基础实操说明
i7j8k9l 仓库初始化空白基础版本
🔹 新手专属避坑提醒
- 完整log日志信息繁杂，新手优先锁定--oneline极简格式，快速上手不迷茫；
- 查看完毕后，按键盘q键一键退出日志界面，回归终端正常操作；
- 版本号为系统自动生成，禁止手动篡改，回溯仅复制前7位字符即可正常使用。

✅ 命令 5：git push —— 本地版本同步推送远程GitHub（云端备份核心）
🔹 核心作用
将本地仓库中所有已commit归档的正式代码版本，安全同步推送至云端GitHub远程仓库，完成云端双重备份，同时支持团队队友、多台办公设备同步调取最新代码，打通本地与远程核心链路。
🔹 新手通用万能标准语法（全程复用无需改动）
git push origin main
名词释义：origin=远程GitHub仓库默认固定别名；main=仓库默认主分支，新手全程无需更换分支。
🔹 极简实操流程
本地完成add归集+commit归档 → 终端执行push推送命令 → 进度100%无报错即推送成功 → 刷新GitHub个人仓库网页，即可同步查看新增文件及完整提交日志。
🔹 高频报错一键排查修复（新手必存）
1. 报错：fatal: 'origin' does not appear to be a git repository
原因：本地仓库未绑定关联远程GitHub仓库
解决方案：提前执行绑定命令 git remote add origin 你的GitHub仓库完整地址，重新推送即可。
2. 报错：rejected: failed to push some refs
原因：云端GitHub网页端有本地未同步的新增修改内容，版本不一致冲突
解决方案：先执行git pull origin main拉取云端最新内容合并，无冲突后再次推送。
3. 弹窗要求输入账号密码：禁止填GitHub登录密码！用户名填注册绑定邮箱，密码填个人PAT令牌（ghp_开头专属密钥）。

✅ 命令 6：git pull —— 拉取远程GitHub最新代码同步本地（多人协作刚需）
🔹 核心作用
一键拉取云端GitHub远程仓库最新归档代码、队友协同修改内容、网页端手动编辑内容，自动合并同步到本地电脑仓库，保证多设备、多人团队本地+云端代码版本完全一致，规避协作冲突。
🔹 新手通用标准语法
git pull origin main
🔹 实战场景演示
打开GitHub仓库网页，直接在线编辑修改README.md文件并提交保存 → 回到本地Git终端 → 执行pull拉取命令 → 执行完成后打开本地README文件，云端修改内容已全自动同步到位。
🔹 新手专属避坑提醒
- 铁律守则：多人协作开发、切换电脑办公、每次开工写代码前，必须先执行pull拉取最新内容；
- 本命令自带「拉取+自动合并」双重功能，仅本地、远程未改同一文件同一行，即可零冲突同步；若出现冲突，暂停修改，优先对照日志排查适配即可。

✅ 命令 7：git clone —— 一键克隆远程仓库到本地（下载开源项目首选）
🔹 核心作用
完整复刻GitHub云端任意公开/私有仓库全量数据，包含所有代码文件、全历史提交版本、分支配置、归档日志，一键下载到本地指定文件夹，适配下载开源项目、多设备同步个人仓库、复刻团队工程场景。
🔹 标准通用语法
git clone 目标GitHub仓库完整HTTPS地址
🔹 分步实操教程
1. 打开目标GitHub仓库主页，点击右上角绿色「Code」按钮，一键复制仓库HTTPS链接；
2. 在电脑桌面/专属工作文件夹，右键打开Git Bash终端；
3. 粘贴链接执行clone命令，等待加载完成，本地自动生成完整仓库文件夹，直接可用。
🔹 新手专属避坑提醒
- 无需提前执行git init初始化，克隆完成自动生成合规Git本地仓库，开箱即用；
- 公开仓库免费直接克隆，无需登录账号；私有仓库克隆，需核验账号+输入PAT专属令牌授权。

✅ 命令 8：git init —— 普通文件夹初始化Git仓库（本地自建仓库专用）
🔹 核心作用
为本地空白普通文件夹，一键赋能Git全套版本管控能力，自动生成隐藏.git核心配置文件夹，所有版本日志、修改记录、仓库密钥全部加密存储于此文件夹内，适配从零自建本地独立项目仓库。
🔹 极简零参数标准语法
git init
🔹 落地实操步骤
本地新建空白自定义项目文件夹 → 进入文件夹内部打开Git Bash → 执行init命令 → 终端提示「Initialized empty Git repository」即初始化成功，可正常使用全部Git命令。
🔹 高危避坑提醒（严禁违规操作）
- 一个项目文件夹仅执行一次init，已生成Git仓库后重复执行，会损坏仓库配置；
- 绝对禁止手动删除、剪切、修改隐藏.git文件夹，一旦损坏，全量版本历史全部丢失，仓库直接报废；
- 新手优先用git clone复刻云端仓库，无需手动init，从源头规避配置报错。

✅ 命令 9：git restore —— 一键撤销未提交修改（改错代码救命神器）
🔹 核心作用
极速撤销本地工作区误写、改错、冗余代码，一键还原文件至上一次commit归档后的纯净状态，适配未归集、未提交的临时失误修改，堪比代码专属「一键撤回」，救急刚需命令。
🔹 两套高频实用语法
1）撤销工作区未add的误改文件（最常用）
git restore 文件名.后缀
2）把已add暂存的文件退回工作区（保留文件内容，仅退出暂存区）
git restore --staged 文件名.后缀
🔹 实战救急场景演示
误改学习笔记.md，写入大量无效垃圾内容并保存 → git status核验文件标红未归集 → 执行restore撤销命令 → 打开文件自动还原纯净版本，误改内容全部清空。
🔹 新手专属避坑提醒
- 仅能撤销未commit归档的临时修改，已提交入库的版本无法用此命令撤回；
- 撤销后误改内容永久无法找回，执行前务必确认无需保留临时修改，谨慎操作。

✅ 命令 10：git reset —— 版本回滚撤销已提交代码（正式版本改错兜底）
🔹 核心作用
精准回溯本地仓库至任意历史归档版本，安全撤销已commit入库、甚至已推送云端的错误正式版本，适配上线前发现bug、提交冗余代码、版本逻辑异常等兜底修复场景。
🔹 新手安全优先语法（零丢代码，强制首选）
git reset --soft 目标版本号前7位
参数释义：--soft=安全保护模式，仅撤销错误commit归档，本地代码文件完整保留，不丢失任何手写内容，新手全程无忧。
🔹 进阶高危语法（新手严禁单独使用）
git reset --hard 目标版本号前7位
风险提示：强制清空所有后续修改代码、销毁临时文件，数据不可逆丢失，仅资深开发者确认废弃代码后可使用。
🔹 安全回滚实操流程
1. 执行git log --oneline查询全量历史版本，复制需要回溯的目标版本前7位编号；
2. 粘贴版本号执行--soft安全回滚命令；
3. 重新查看日志，已自动跳转至目标纯净版本，本地代码完好无损，可重新修改合规提交。
🔹 职场&新手双重避坑提醒
- 个人私有仓库回滚后，可搭配git push -f origin main强制同步云端；
- 多人协作团队公用仓库，严禁使用强制推送，会直接覆盖队友正常提交代码，造成团队工程故障；
- 全程坚守：新手只用--soft，坚决不用--hard，杜绝数据丢失风险。
